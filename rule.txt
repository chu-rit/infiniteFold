[Project: Infinite Fold] - 4x4 Logic Game
1. Game Overview
Concept: A puzzle game where players fold a 4x4 grid to merge identical numbers based on symmetry.

Goal: Merge numbers to reach higher values without getting "Deadlocked" (no more possible moves).

2. Core Mechanics: Folding (The "Fold" Action)
The player can fold the board from any of the 4 edges (Top, Bottom, Left, Right).

Partial Fold (1-Row): Folds only the outermost row/column onto the next one.

Example (Top to Bottom): Row 0 folds onto Row 1. (Rows 2, 3 remain still).

Full Fold (2-Row / Half): Folds the board exactly in half.

Example (Top to Bottom): Row 0 folds onto Row 3, Row 1 folds onto Row 2.

3. Logic Rules (Critical for AI)
When a fold is attempted, the system must check the "Target Cell" for every "Source Cell":

Merge: If Source == Target, they merge into Source * 2.

Move: If Target == 0 (Empty), the Source value moves to the Target.

Block (Mismatch): If Source != Target and Target != 0, the entire fold is invalid.

Execution: A fold only executes if all moving cells meet the Merge or Move conditions.

4. Game Loop & Spawning
Trigger: A new number spawns only if the fold resulted in zero merges.

Combo Reward: If at least one pair was merged, skip spawning a new number for that turn.

Dynamic Balancing:

High Space: Spawn numbers that don't exist on the board to increase difficulty.

Low Space (< 3 cells): Spawn the most frequent number on the board to help the player.

5. UI/UX Requirements
Interaction: Input via swipe. Swipe distance determines Fold Depth (Short = 1-Row, Long = 2-Row).

Real-time Preview: Show ghost numbers on Target Cells during the swipe/drag.

Visual Feedback:

Show merged results in a semi-transparent state.

If a fold is blocked by a Mismatch, highlight the offending cells in Red.

6. Technical Implementation Roadmap
Data Structure: 4x4 2D Array (Integer).

Symmetry Function: Create a function getSymmetricIndex(index, foldDepth) to calculate target coordinates.

Validation: A canFold(direction, depth) function to check for Mismatches before updating the state.

GameOver: Check every possible move (4 directions * 2 depths = 8 total options). If all return false, Game Over.
